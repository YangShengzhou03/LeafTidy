# 轻羽归档 LeafTidy

一个安静的文件整理工具。

你有没有遇到过这样的情况：下载了一堆照片，文件名乱七八糟；备份文件夹里重复文件到处都是；想整理照片却发现拍摄时间全是错的？

LeafTidy 就是来解决这些问题的。它是一款纯离线的桌面应用，不需要联网，你的文件不会上传到任何地方。

---

## 它能做什么

### 文件整理

最基础的功能。你可以把多个文件夹里的文件，按照类型、日期或者大小，整理到一个统一的目录里。比如把所有 JPG 图片放到"图片"文件夹，所有 PDF 文档放到"文档"文件夹。

如果你想保留原来的目录结构，也可以选择"保留原目录结构"模式，这样只是把文件搬运过去，不会打乱原有的组织方式。

### 批量重命名

给一堆文件改名，不用一个个手动改。你可以用模板，比如 `{日期}_{原名}`，就会变成 `2024-01-15_照片.jpg`。序号位数可以自定义，想三位数就三位数，想五位数就五位数。

### 重复文件清理

用 MD5 校验找出重复文件。可以选择只删完全一样的，或者大小一样但内容可能不同的也删。删之前会给你看列表，确认后再动手。

### 照片相关

如果你经常拍照，这几个功能可能有用：

- **修复拍摄时间**：有些照片的 EXIF 时间是错的，可以用文件修改时间或者你自己指定的时间来修正
- **照片脱敏**：上传照片之前，把 GPS 位置、相机型号这些隐私信息去掉
- **GPS 位置分类**：根据照片里的 GPS 信息，自动按省/市/区县分类。用的是离线地名库，不需要联网
- **AI 智能分类**：本地 AI 模型识别照片内容，人像、风景、宠物、美食这些类别自动归类

### 其他

- **文件筛选**：按类型、大小、时间过滤文件
- **附属文件清理**：删掉那些系统自动生成的 Thumbs.db、.DS_Store、desktop.ini
- **混合规则归档**：年份/月份/省份/AI 标签多层组合，想怎么组织就怎么组织

### 日志和回滚

每次操作都会记录日志。如果整理完发现搞错了，可以从日志里找到那批操作，一键回滚。移动和重命名操作都支持回滚。

## 技术栈

前端用的是 Vue 3 + TypeScript + Element Plus，后端是 Rust，桌面壳是 Tauri 2。整个应用打包后体积比较小，启动也快。

主要的 Rust 依赖：

- `kamadak-exif` — 解析照片 EXIF 信息
- `walkdir` — 递归扫描目录
- `chrono` — 时间处理
- `md-5` — 文件哈希计算
- `trash` — 删除文件时先放到回收站，安全一点

---

## 怎么运行

需要 Node.js 18 以上和 Rust 1.77 以上。

```bash
# 安装依赖
npm install

# 开发模式运行
npm run dev

# 打包
npm run build:tauri
```

## 项目结构

```
leaf-tidy/
├── src/                    # 前端代码
│   ├── components/         # 左侧栏、右侧栏
│   ├── composables/        # 文件操作、布局状态管理
│   ├── views/              # 各功能页面
│   └── App.vue             # 主布局
├── src-tauri/              # Rust 后端
│   └── src/
│       ├── commands.rs     # Tauri 命令
│       ├── file_ops.rs     # 文件读写移动
│       ├── metadata.rs     # EXIF/GPS 处理
│       └── log.rs          # 日志和回滚
└── README.md
```
