## 项目简介

LeafMaster（轻羽大师版）是轻羽信息自动系统的专业版本，基于 Tauri + Vue 3 + Element Plus 技术栈开发，追求极致的性能和稳定性。

## 核心特性

- **极低内存占用**: ~20MB 内存，比普通版降低 87%
- **秒级启动速度**: 基于 Rust 和 Tauri 的原生性能
- **现代化界面**: Element Plus UI 组件库，用户体验优秀
- **稳定可靠**: 完善的错误处理和日志记录机制

## 功能特性

### 核心功能

- **定时发送**: 支持精确到秒的定时消息和文件发送
- **间隔循环**: 按间隔时间循环执行任务
- **文件发送**: 支持发送各种格式的文件（PDF、Word、Excel、图片等）
- **消息管理**: 发送文本消息到指定联系人或群组
- **智能拆句**: 自动拆分长消息，避免消息过长
- **Excel 导入导出**: 批量管理任务配置

### 高级功能

- **多任务并发**: 同时执行多个任务
- **任务队列管理**: 智能调度任务执行顺序
- **失败重试机制**: 自动重试失败的任务
- **实时日志监控**: 查看任务执行详情
- **网络时间同步**: 确保定时任务精准执行
- **防休眠机制**: 保持系统唤醒状态

## 定时原理

### 任务调度机制

LeafMaster 采用基于时间轮询的定时任务调度机制：

1. **时间检查循环**
   - 系统启动后，每隔 1 秒检查一次所有任务
   - 对比当前时间与任务的下次执行时间
   - 当当前时间 >= 执行时间时，触发任务执行

2. **任务优先级**
   - 所有到期的任务按照创建顺序依次执行
   - 单任务串行执行，确保稳定性和可追溯性
   - 避免多任务并发导致的资源竞争和微信窗口冲突

3. **时间精度**
   - 理论精度：±1 秒
   - 实际精度：±2-3 秒（考虑系统负载和微信响应时间）
   - 建议设置任务时间时提前 1-2 分钟

### 多任务处理策略

当多个任务设定在同一时间执行时：

1. **顺序执行**
   - 按任务创建时间排序
   - 依次执行每个任务
   - 每个任务完成后才执行下一个

2. **执行时间计算**
   ```
   任务 A：设定时间 10:00:00
   任务 B：设定时间 10:00:00
   任务 C：设定时间 10:00:00

   实际执行：
   10:00:00 - 任务 A 开始执行
   10:00:03 - 任务 A 完成（耗时3秒）
   10:00:03 - 任务 B 开始执行
   10:00:06 - 任务 B 完成（耗时3秒）
   10:00:06 - 任务 C 开始执行
   10:00:09 - 任务 C 完成（耗时3秒）
   ```

3. **优化建议**
   - 如果有大量任务需要同一时间执行，建议分散设置时间
   - 例如：每隔 5 分钟设置一个任务，避免堆积
   - 系统会自动处理时间延迟，但可能影响后续任务的准时性

## 发送流程

### 消息发送完整流程

1. **微信窗口检测**
   - 扫描系统窗口，查找微信主窗口（ClassName: `mmui::MainWindow`）
   - 如果微信未运行或未登录，返回错误

2. **窗口激活**
   - 如果微信最小化，还原窗口
   - 将微信窗口居中到屏幕中央
   - 置顶并激活微信窗口
   - 等待 500ms 确保窗口完全激活

3. **定位联系人**
   - 检查当前聊天窗口标题是否匹配目标联系人
   - 如果匹配，直接进入聊天界面
   - 如果不匹配，在会话列表中搜索
   - 如果会话列表中找不到，使用微信搜索功能
   - 在搜索结果中匹配联系人、群聊等
   - 点击联系人进入聊天界面

4. **输入消息**
   - 定位聊天输入框（AutomationId: `chat_input_field`）
   - 使用 UI Automation 设置输入框内容
   - 等待 200ms 确保输入完成

5. **发送消息**
   - 查找发送按钮（Name: "发送", ClassName: `mmui::XOutlineButton`）
   - 如果发送按钮可用，点击发送
   - 如果发送按钮不可用，模拟回车键发送
   - 等待 200ms 确保发送完成

6. **结果返回**
   - 成功：返回 `{success: true, message: "已发送给 XXX"}`
   - 失败：返回 `{success: false, message: "发送失败: 错误详情"}`

### 文件发送流程

1. **文件检查**
   - 验证文件路径是否存在
   - 获取文件绝对路径

2. **复制到剪贴板**
   - 构造 DROPFILES 结构体
   - 将文件路径写入剪贴板
   - 等待 100ms 确保剪贴板就绪

3. **定位联系人**
   - 同消息发送流程

4. **粘贴文件**
   - 点击聊天输入框
   - 模拟 Ctrl+V 粘贴文件
   - 等待 500ms 确保文件加载

5. **发送文件**
   - 同消息发送流程

6. **清理剪贴板**
   - 清空剪贴板内容
   - 避免影响后续操作

## 错误处理机制

### 错误类型

1. **微信未连接**
   - 检测不到微信窗口
   - 微信未登录状态

2. **联系人未找到**
   - 联系人名称错误
   - 联系人不在通讯录中
   - 搜索结果匹配失败

3. **控件定位失败**
   - 发送按钮不可用
   - 输入框未找到
   - 窗口元素获取失败

4. **发送失败**
   - 网络异常
   - 微信无响应
   - 权限不足

### 错误处理流程

1. **后端捕获**
   - 所有错误在 Rust 后端捕获
   - 记录详细错误日志到文件
   - 返回友好的错误信息

2. **前端处理**
   - 接收错误信息
   - 记录错误日志到文件
   - 回退执行次数（`executeCount--`）
   - 标记任务状态为 `failed`
   - 播放错误提示音

3. **日志记录**
   - 每个错误都记录到日志文件
   - 包含时间戳、模块、错误详情
   - 可在"日志分析"页面查看

### 失败重试策略

- **不自动重试**: 为避免重复发送，失败任务不会自动重试
- **手动重试**: 用户可以手动启用任务，再次执行
- **执行次数保护**: 失败后回退执行次数，确保下次执行正确

## 日志系统

### 日志级别

- **INFO**: 正常操作日志（任务开始、完成等）
- **WARN**: 警告日志（微信未连接等）
- **ERROR**: 错误日志（任务失败、发送异常等）
- **DEBUG**: 调试日志（仅在开发模式）

### 日志文件

- 存储位置：`%LOCALAPPDATA%\LeafMaster\logs\`
- 文件命名：`leaf-master-YYYY-MM-DD.log`
- 自动按日期分割
- 可在"日志分析"页面查看、下载、删除

### 日志内容

每条日志包含：
- 时间戳（精确到毫秒）
- 日志级别（INFO/WARN/ERROR/DEBUG）
- 模块名称（WeChat/TaskExecutor 等）
- 详细信息

## 快速开始

### 系统要求

- **操作系统**: Windows 10 64位或更高版本
- **微信版本**: 3.9.0 及以上版本
- **内存**: 建议 8GB RAM（推荐 16GB）
- **存储空间**: 100MB 可用空间

### 安装使用

1. **下载安装包**
   - 从 [Gitee 发行版](https://gitee.com/Yangshengzhou/LeafAuto/releases) 下载最新版本

2. **安装程序**
   - 双击安装包，按照向导完成安装

3. **登录微信**
   - 确保微信已登录并保持运行状态
   - 建议将微信窗口保持在桌面上

4. **创建任务**
   - 打开 LeafMaster，进入"自动信息"
   - 创建新任务，设置发送内容、接收对象和时间
   - 启用任务，点击"开始执行"

### 使用技巧

- 建议给联系人设置 5 个纯汉字以内的备注，避免特殊字符
- 设置任务时间时，建议提前 1-2 分钟，避免网络延迟
- 定期备份任务配置到 Excel 文件
- 定期清理已完成的任务，保持列表整洁
- 查看日志文件了解任务执行详情

## 开发指南

### 技术栈

- **前端**: Vue 3 + Element Plus + TypeScript
- **后端**: Rust + Tauri
- **状态管理**: Pinia
- **构建工具**: Vite

### 本地开发

```bash
# 克隆仓库
git clone https://gitee.com/Yangshengzhou/LeafAuto.git
cd LeafAuto
git checkout leaf-master

# 安装依赖
npm install

# 开发模式运行
npm run start

# 构建生产版本
npm run build
```
